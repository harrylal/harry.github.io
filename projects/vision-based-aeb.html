Vision-Only Reinforcement Learning AEB

Technical Overview

The Vision-Based Autonomous Emergency Braking (AEB) system replaces traditional radar-dependent architectures with a monocular vision pipeline coupled to a Proximal Policy Optimization (PPO) reinforcement learning agent.

Perception Pipeline

The perception stack runs a multi-scale object detector optimized with TensorRT on NVIDIA Jetson hardware. Detected objects are tracked across frames using a Kalman filter-based tracker, providing stable bounding boxes and velocity estimates for risk computation.

Risk Prediction

Rather than using fixed distance thresholds, the system learns collision risk scores from simulated and real-world driving scenarios. The PPO agent optimizes intervention timing by balancing:

  • Safety recall — intervening before collisions occur
  • Driver comfort — minimizing unnecessary braking events
  • Latency constraints — acting within the 80ms end-to-end budget

Deployment Architecture

Camera → Detection → Tracking → Risk Model → PPO Policy → CAN Bus → Brake Actuator

Each stage is profiled and optimized independently, with the PPO policy running as a lightweight inference call (< 5ms) after risk features are computed.

Key Results

The system achieved 94% collision detection recall with a 3× reduction in false positives compared to the previous rule-based approach, validated across 10,000+ hours of fleet driving data.
